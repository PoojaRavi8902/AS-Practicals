What is computational thinking?

Computational thinking is a logical process for solving problems so the solution can be understood by humans and implemented by computers. Key strands: Abstraction, Decomposition, Data modelling, Pattern recognition, Algorithmic thinking.

Quick classroom activity
  1. Show students a complex real-world object (e.g. phone plan). Ask them to list only details needed to decide which plan to choose — that demonstrates abstraction.
  2. Give a complex task (e.g. organising a school trip) and have groups decompose it into modules (transport, food, permissions).

Abstraction — filter out what is not essential

Abstraction means showing only the essential information needed for a task. Example in the book: the London Underground map. The map is abstract because it shows stations and connections, not exact geographical details.

Use the controls to remove/add detail and see how abstraction makes decision-making easier.

Decomposition & program modules

Decomposition splits a big problem into smaller, manageable sub-problems (modules). Each module can be designed, tested and reused independently. In programming, modules become procedures, functions or classes.

Example: school trip (decompose)
  • Transport — choose vendor, arrange timing
  • Permissions — collect forms and payments
  • Itinerary — places to visit, timing
  • Safety — first aid, emergency contacts
Programming example: 
MAIN
  GetTripDetails()
  ArrangeTransport()
  CollectPermissions()
  RunItinerary()
END

Procedure ArrangeTransport()
  BookBus()
  ConfirmSeats()
EndProcedure

What is an algorithm?

An algorithm is a sequence of defined steps to perform a task. Everyday examples include recipes and instruction manuals. Algorithms must be correct, complete and in a logical order.

Key properties of a good algorithm

  • Correctness — produces correct outputs for valid inputs
  • Clarity — easy to read and understand
  • Efficiency — uses memory and time sensibly
  • Generality — works for a class of inputs
  • Terminating — finishes in finite time for valid inputs
Activity — ordering steps

Give students scrambled algorithm steps (e.g. car wheel change). Ask them to reorder — discuss why an incorrect order fails.

Constructs & notations: Structured English, Pseudocode, Flowcharts

We express algorithms using structured English, pseudocode or flowcharts. All three represent the same control constructs: assignment, sequence, selection, repetition.

Assignment / Sequence 
Pseudocode:
A ← 34
B ← B + 1
Selection 
IF A > B THEN
  OUTPUT A
ELSE
  OUTPUT B
ENDIF
Repetition 
REPEAT
  INPUT X
UNTIL X = 0

Flowcharts use shapes: oval (start/end), rectangle (process), diamond (decision), parallelogram (input/output). Encourage students to draw both pseudocode and flowchart for the same algorithm.

13.05 Two-dimensional Arrays

  
When we write a table of data (a matrix) on paper and want to refer to individual elements, the convention is to give the row number first and then the column number. In programming, this same idea is applied when declaring and using two-dimensional arrays.


  

    
Diagram: 2D Array Representation

    
      2D Array (3x3)
      [ [1, 2, 3],
        [4, 5, 6],
        [7, 8, 9] ]

      Access element [row 2, col 3] → 6
    

  


  

    
Example in Pseudocode

    
      DECLARE table : ARRAY[1:3, 1:3] OF INTEGER
      FOR row ← 1 TO 3
          FOR col ← 1 TO 3
              table[row, col] ← row * col
          ENDFOR
      ENDFOR

      OUTPUT table[2, 3]   // Displays 6

    2. 
      
  2. Write pseudocode to find the highest individual mark in the array.
    3. 
    

    

      Worked Answer:
      
      // Part (a)
      FOR student ← 1 TO 4
          total ← 0
          FOR subject ← 1 TO 3
              total ← total + scores[student, subject]
          ENDFOR
          OUTPUT "Student ", student, ": ", total
      ENDFOR

      // Part (b)
      max ← scores[1,1]
      FOR student ← 1 TO 4
          FOR subject ← 1 TO 3
              IF scores[student, subject] > max THEN
                  max ← scores[student, subject]
              ENDIF
          ENDFOR
      ENDFOR
      OUTPUT "Highest mark: ", max
      

    

  





  
12.08 Stepwise Refinement

  
When problems are large and complex, solving them in one go can be overwhelming. To make them manageable, we use Stepwise Refinement. This involves breaking a big problem into smaller steps, then breaking those steps down further until each one is simple enough to implement with basic constructs: sequence, assignment, selection, repetition, input, and output.


  

    
Diagram: Stepwise Refinement

    
    Big Problem
        ↓
    Sub-Problem 1   Sub-Problem 2   Sub-Problem 3
        ↓                ↓                ↓
    Step 1.1  Step 1.2   Step 2.1 ...   Step 3.1 ...
    

  


  

    
Worked Example 12.10: Drawing a Pyramid

    
Problem: Input a symbol and an odd number. Output a pyramid with that symbol, where the final row has the given number of symbols.

    
Example Input: Symbol = A, Max = 9

    
        A
       AAA
      AAAAA
     AAAAAAA
    AAAAAAAAA
    

    
First Attempt (Structured English):

    
    01 Set up initial values
    02 REPEAT
    03     Output spaces
    04     Output symbols
    05     Adjust values for next row
    06 UNTIL final row is reached
    


    
Refined Pseudocode:

    
    01.1 INPUT Symbol
    01.2 INPUT MaxNumberOfSymbols (must be odd)
    01.3 NumberOfSpaces ← (MaxNumberOfSymbols – 1) / 2
    01.4 NumberOfSymbols ← 1

    02 REPEAT
       03.1 FOR i ← 1 TO NumberOfSpaces
               OUTPUT Space
             NEXT i
       04.1 FOR i ← 1 TO NumberOfSymbols
               OUTPUT Symbol
             NEXT i
       04.4 OUTPUT Newline
       05.1 NumberOfSpaces ← NumberOfSpaces – 1
       05.2 NumberOfSymbols ← NumberOfSymbols + 2
    UNTIL NumberOfSymbols > MaxNumberOfSymbols

Worked Example 12.11: Pyramid with Modules

    
Instead of writing one long solution, we can break the pyramid problem into modules (procedures and functions).

    
    CALL SetValues
    REPEAT
        CALL OutputSpaces
        CALL OutputSymbols
        CALL AdjustValues
    UNTIL NumberOfSymbols > MaxNumberOfSymbols

    PROCEDURE SetValues
        INPUT Symbol
        MaxNumberOfSymbols ← ValidatedMaxNumber
        NumberOfSpaces ← (MaxNumberOfSymbols - 1) / 2
        NumberOfSymbols ← 1
    ENDPROCEDURE

    FUNCTION ValidatedMaxNumber RETURNS INTEGER
        REPEAT
            INPUT MaxNumber
        UNTIL MaxNumber MOD 2 = 1
        RETURN MaxNumber
    ENDFUNCTION
    


    
Each module can be reused in other problems, making this approach efficient and organized.

  


  

    
Worked Example 12.13: Pyramid with Parameters

    
Passing values into procedures makes them independent and reusable:

    
    PROCEDURE OutputSpaces(NumberOfSpaces)
        FOR i ← 1 TO NumberOfSpaces
            OUTPUT Space
        NEXT i
    ENDPROCEDURE

    PROCEDURE OutputSymbols(NumberOfSymbols, Symbol)
        FOR j ← 1 TO NumberOfSymbols
            OUTPUT Symbol
        NEXT j
        OUTPUT Newline
    ENDPROCEDURE
    

  


  

    
Key Terms

    
    
      
  • Stepwise Refinement: Breaking a problem into smaller and smaller steps until implementable.
    • 
      
  • Module: A reusable block of code (procedure or function).
    • 
      
  • Procedure: A sequence of steps grouped under a name, does not return a value.
    • 
      
  • Function: A sequence of steps grouped under a name, returns a value.
    • 
      
  • Local Variable: Accessible only within the module.
    • 
      
  • Global Variable: Accessible across all modules.
