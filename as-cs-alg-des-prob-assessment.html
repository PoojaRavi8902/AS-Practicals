Part A — Multiple Choice Questions (20)

1. Which loop guarantees at least one execution?

Answer: C — REPEAT...UNTIL executes body before testing.

2. In pseudocode the assignment is denoted by:

Answer: C — the arrow (←) denotes assignment.

3. Which is NOT a basic construct (chapter)?

Answer: C — recursion is not listed as a basic construct in the chapter.

4. What is a rogue value?

Answer: B — a rogue value terminates a sequence; must be of same type but outside normal range.

5. Which flowchart shape is for decisions?

Answer: C — Diamond indicates decision/condition.

6. Which loop is count-controlled?

Answer: C — FOR...NEXT is count-controlled (counter managed automatically).

7. Why use BiggestSoFar method?

Answer: B — uses fewer variables and adapts to more numbers easily.

8. Which operator yields remainder?

Answer: B — MOD returns the remainder.

9. What does an identifier table document?

Answer: A — lists identifiers, explanations and data types.

10. What is stepwise refinement?

Answer: A — refine steps until they map to basic constructs.

11. Which is an example of abstraction?

Answer: A — tube map is abstraction by removing unnecessary detail.

12. Local variables are:

Answer: A — local variables are module-scoped.

13. Which pseudocode statement outputs data?

Answer: B — OUTPUT prints messages or variables.

14. If REPEAT...UNTIL condition is true initially, what happens?

Answer: A — REPEAT executes body at least once before testing.

15. Identifier NumberOfGuesses follows which style?

Answer: B — CamelCaps (each word capitalised, no spaces).

16. Which loop suits reading until rogue value?

Answer: B — REPEAT...UNTIL (or WHILE) is used; REPEAT is common for rogue-value sequences.

17. What does Count ← Count + 1 do?

Answer: B — increases Count by one.

18. A function differs from a procedure because:

Answer: A — functions return a value used in expressions.

19. Outer loop for printing R rows and C columns is:

Answer: B — the row loop is outermost; inner loop handles columns.

20. Why prefer local variables?

Answer: B — local variables reduce side effects and make modules reusable.

1. Define Abstraction.

Model answer: Abstraction is filtering out non-essential details and representing only the necessary information for solving a problem.

Model answer: Decomposition is breaking a large problem into smaller sub-problems or modules that are easier to solve.

Model answer:

INPUT B
INPUT C

  IF A > C THEN OUTPUT A ELSE OUTPUT C ENDIF

  IF B > C THEN OUTPUT B ELSE OUTPUT C ENDIF

9. Why use REPEAT...UNTIL for rogue value sequences?

Model answer: REPEAT...UNTIL executes body first ensuring at least one input is read; suitable for sequences terminated by a rogue value.

10. Validate odd input N — pseudocode.

  INPUT N

Model answer: A procedure is a named sequence of steps performing a sub-task (e.g., PROCEDURE OutputSymbols(Number, Symbol)).

14. Print R x C grid of '*' using nested loops (pseudocode).

15. What is the benefit of modules?

Q1. Modulo-11 check digit (flowchart → pseudocode). Leftmost weight 10, nine digits. If check digit = 10 → 'X'.

Weighting ← 10
Total ← 0
Count ← 1
REPEAT
  INPUT Digit
  Value ← Digit * Weighting
  Total ← Total + Value
  Weighting ← Weighting - 1
  Count ← Count + 1
UNTIL Count > 9
Remainder ← Total MOD 11
CheckDigit ← 11 - Remainder
IF CheckDigit = 10 THEN
  CheckDigit ← 'X'
ELSEIF CheckDigit = 11 THEN
  CheckDigit ← 0
ENDIF
OUTPUT CheckDigit

Worked example: Digits 0..8 → sum 156; remainder 2; check digit 9.

Q2. REPEAT until number is zero — count positives and negatives. Write pseudocode.

PositiveCount ← 0
NegativeCount ← 0
REPEAT
  INPUT Number
  IF Number > 0 THEN
    PositiveCount ← PositiveCount + 1
  ELSEIF Number < 0 THEN
    NegativeCount ← NegativeCount + 1
  ENDIF
UNTIL Number = 0
OUTPUT PositiveCount
OUTPUT NegativeCount

Worked example: Inputs 5,-3,7,-1,0 → Positive=2, Negative=2.

Q3. Using WHILE, read numbers until rogue value -1; output average if count > 0.

RogueValue ← -1
Total ← 0
Count ← 0
INPUT Number
WHILE Number <> RogueValue DO
  Count ← Count + 1
  Total ← Total + Number
  INPUT Number
ENDWHILE
IF Count > 0 THEN
  Average ← Total / Count
  OUTPUT Average
ELSE
  OUTPUT "No valid numbers entered"
ENDIF

Worked example: Inputs 10,20,-1 → Average = 15.

Q4. Stepwise refinement for a hollow triangle (Task 12.06). Provide top-level and OutputRow procedure (params: NumberOfSpaces, NumberOfSymbols, MaxNumber).

CALL SetValues
REPEAT
  CALL OutputRow(NumberOfSpaces, NumberOfSymbols, MaxNumberOfSymbols, Symbol)
  CALL AdjustValuesForNextRow
UNTIL NumberOfSymbols > MaxNumberOfSymbols

PROCEDURE OutputRow(NumberOfSpaces, NumberOfSymbols, MaxNumberOfSymbols, Symbol)
  FOR i ← 1 TO NumberOfSpaces
    OUTPUT Space // no newline
  NEXT i
  IF NumberOfSymbols = MaxNumberOfSymbols THEN
    FOR j ← 1 TO NumberOfSymbols
      OUTPUT Symbol
    NEXT j
  ELSE
    OUTPUT Symbol
    IF NumberOfSymbols > 1 THEN
      FOR k ← 1 TO NumberOfSymbols - 2
        OUTPUT Space
      NEXT k
      OUTPUT Symbol
    ENDIF
  ENDIF
  OUTPUT Newline
ENDPROCEDURE

Notes: Validate MaxNumberOfSymbols is odd using REPEAT...UNTIL with MOD 2 = 1.

Q5. Function AdjustedNumberOfSymbols(NumberOfSymbols) returns NumberOfSymbols+2 and usage in top-level.

FUNCTION AdjustedNumberOfSymbols(NumberOfSymbols) RETURNS INTEGER
  NumberOfSymbols ← NumberOfSymbols + 2
  RETURN NumberOfSymbols
ENDFUNCTION

// Call in top-level:
NumberOfSymbols ← AdjustedNumberOfSymbols(NumberOfSymbols)

Explanation: Functions return values to caller; using functions keeps code modular and testable.
